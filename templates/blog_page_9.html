딥러닝 학습 방법 완벽 가이드

다양한 학습 패러다임과 방법론

딥러닝의 다양한 학습 방법들을 체계적으로 이해할 수 있습니다

안녕하세요! 오늘은 딥러닝 학습 방법에 대해 자세히 알아보겠습니다. 딥러닝에는 다양한 학습 패러다임과 방법론이 존재하며, 각각의 특징과 활용 분야가 다릅니다.

1. 학습/추론 패러다임

Zero-shot Learning

정의: 학습 데이터에 없는 태스크도 프롬프트만으로 수행합니다.

특징: LLM 시대에 핵심 특징이며, 일반화 능력을 강조합니다.

예시: '얼룩말'을 학습하지 않았지만, '말처럼 생기고 줄무늬 있음'을 통해 추론

Zero-shot Learning의 개념

Few-shot Learning

정의: 소량의 예시만 보고 새로운 태스크를 수행합니다.

활용: Prompt Engineering으로 자주 활용되며, 얼굴 인식, OCR 등에 사용됩니다.

대표 모델: Matching Network, Prototypical Network, GPT with prompt

Few-shot Learning의 예시

Instruction Tuning

정의: 자연어 지시문을 이해하고 수행할 수 있도록 추가 학습합니다.

특징: ChatGPT류 모델 기반으로 사용되며, 다양한 태스크에 대한 지시를 이해할 수 있습니다.

RLHF (Reinforcement Learning with Human Feedback)

정의: 사람 피드백 기반 보상 모델을 통해 안전하고 유용한 답변을 생성합니다.

활용: OpenAI, Anthropic 등이 활용하며, AI의 안전성과 유용성을 동시에 확보합니다.

2. 파인튜닝/경량화

LoRA (Low-Rank Adaptation)

정의: 대형 모델 파라미터 전체 학습 대신, 저차원 행렬만 학습하여 효율적 미세조정을 수행합니다.

장점: GPU/메모리 절감, 도메인 특화 모델에 활용 가능합니다.

LoRA의 저차원 적응 구조

PEFT (Parameter-Efficient Fine-Tuning)

정의: LoRA, Prefix Tuning, Adapter 등을 포함하는 개념입니다.

특징: 적은 파라미터로 특정 도메인 맞춤화가 가능합니다.

Quantization (양자화)

정의: 가중치를 저정밀도(INT8, FP16 등)로 변환하여 속도/메모리를 절감합니다.

활용: Edge/온디바이스 모델에 활용됩니다.

Pruning (가지치기)

정의: 중요도 낮은 파라미터를 제거하여 경량화, 추론속도를 개선합니다.

Distillation (지식 증류)

정의: 큰 모델의 지식을 작은 모델(Student)에 전수합니다.

목적: 성능-효율의 절충을 달성합니다.

3. 멀티모달/확장

Multimodal AI

정의: 텍스트+이미지+음성+영상 통합 처리 (예: GPT-4o, Gemini, Claude Sonnet)

활용: 의료 영상 분석, 음성 비서, 비전-언어 이해에 활용됩니다.

멀티모달 AI의 다양한 데이터 처리

RAG (Retrieval-Augmented Generation)

정의: 검색+생성 결합으로 최신/외부 데이터를 반영합니다.

장점: "Hallucination 줄이고 신뢰성↑"을 달성합니다.

Agentic AI (AI Agent)

정의: LLM이 도구/환경을 제어하며 계획·실행합니다.

활용: AutoGPT, LangChain 기반 워크플로우에 사용됩니다.

4. 모델 아키텍처/인프라

Transformer 최적화

개념: Attention 메커니즘 효율화 (FlashAttention 등)

목적: 더 큰 컨텍스트 윈도우를 지원합니다.

Mixture of Experts (MoE)

정의: 일부 전문가 네트워크만 활성화하여 연산 효율+성능을 확보합니다.

Sovereign AI / On-device AI

개념: 국가/기업/개인 데이터 주권·보안을 강화하는 트렌드입니다.

활용: 스마트폰·엣지 디바이스 탑재 모델에 사용됩니다.

5. 책임/윤리·보안

RAI (Responsible AI)

정의: 편향(Bias), 독성, 개인정보 보호를 대응합니다.

중요성: 기업/정부 AI 규제 대응의 핵심입니다.

Watermarking / Model Security

정의: AI 생성물 판별, 저작권·위조를 방지합니다.

목적: AI Output 검증 트렌드입니다.

AI 보안과 윤리의 중요성
