테이블 데이터 증강 기법

구조화된 데이터에서 모델 성능을 향상시키는 방법

테이블 데이터 증강은 구조화된 데이터의 다양성을 확보하는 중요한 기술입니다

안녕하세요! 오늘은 테이블 데이터 증강(Tabular Data Augmentation)에 대해 자세히 알아보겠습니다. 구조화된 데이터에서도 증강 기법을 통해 모델의 성능을 크게 향상시킬 수 있습니다.

테이블 데이터 증강의 필요성

구조화된 데이터의 특징: 테이블 형태의 데이터는 수치형, 범주형 변수들이 체계적으로 정리되어 있어, 텍스트나 이미지와는 다른 증강 기법이 필요합니다.

Noise Injection (노이즈 주입)

정의: 수치형 변수에 작은 노이즈를 추가해 모델의 Robustness를 높입니다.

특징: Regression, Classification 등에서 과적합을 방지하며, 데이터 분포를 고려하여 적절한 크기의 노이즈를 삽입합니다.

노이즈 주입을 통한 데이터 증강 과정

📝 노이즈 주입 예시

원본 데이터: 나이: 25, 소득: 50000

증강 후: 나이: 25.3, 소득: 50120

→ 작은 노이즈를 추가하여 데이터 다양성 확보

SMOTE (Synthetic Minority Oversampling Technique)

정의: 소수 클래스 샘플을 최근접 이웃(KNN 기반)으로 합성하여 데이터 불균형을 해결합니다.

특징: 분류(Classification) 문제에서 자주 활용되며, 소수 클래스의 샘플을 인위적으로 늘려 균형을 맞춥니다.

SMOTE를 통한 소수 클래스 샘플 생성 과정

📝 SMOTE 예시

문제: 부정 거래(1%) vs 정상 거래(99%)

해결: 부정 거래 샘플을 KNN 기반으로 합성하여 비율 조정

ADASYN (Adaptive Synthetic Sampling)

정의: 결정 경계(Decision Boundary) 근처의 어려운 샘플을 더 많이 생성합니다.

특징: 데이터 불균형 문제를 적응적으로 해결하며, 분류가 어려운 영역에 집중하여 샘플을 생성합니다.

ADASYN의 적응적 샘플링 과정

Undersampling / Oversampling

정의: 다수 클래스 데이터 수를 줄이거나 소수 클래스를 복제하여 데이터 균형을 맞춥니다.

특징: Oversampling은 과적합 위험이 있으므로 주의가 필요하며, 모든 클래스의 분포를 동일하게 만드는 것이 항상 최적은 아닙니다.

Undersampling과 Oversampling의 차이점

테이블 데이터 증강의 장점

  • 데이터 불균형 해결: 소수 클래스 문제를 효과적으로 해결
  • 모델 Robustness 향상: 노이즈에 대한 내성 강화
  • 과적합 방지: 다양한 데이터로 일반화 성능 향상
  • 비용 효율성: 추가 데이터 수집 없이 성능 개선
