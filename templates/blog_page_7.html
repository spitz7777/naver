데이터 전처리는 머신러닝 성공의 핵심 요소입니다

안녕하세요! 오늘은 데이터 전처리(Data Preprocessing)에 대해 자세히 알아보겠습니다. 머신러닝 모델의 성능을 크게 좌우하는 중요한 단계입니다.

데이터 전처리의 중요성

데이터 전처리란: 원본 데이터를 머신러닝 모델이 학습하기 좋은 형태로 변환하는 과정입니다. 이 과정에서 데이터의 품질을 향상시키고 모델의 성능을 최적화할 수 있습니다.

스케일링이 필요한 알고리즘

거리 기반, 선형 모델, 정규분포 가정:

  • KNN: 거리 기반 → Min-Max, Standardization 필요
  • SVM: 거리/내적 기반 → Min-Max, Standardization 필요
  • K-means: 거리 기반 → Min-Max, Standardization 필요
  • PCA: 분산/공분산 기반 → Standardization 필요
  • 신경망: Gradient 안정화 위해 Min-Max, Standardization, Robust Scaling 등 필요

스케일링이 필요한 알고리즘들의 분류

스케일에 덜 민감한 알고리즘

트리 기반, 규칙 분할:

  • Decision Tree: 값의 순서/조건 분할 기반 → Scaling 불필요
  • Random Forest: 트리 기반 → Scaling 불필요
  • XGBoost: 트리 기반 → Scaling 불필요
  • LightGBM: 트리 기반 → Scaling 불필요

트리 기반 알고리즘은 스케일링이 불필요합니다

이상치 탐지 및 처리

탐지 기준

  • Z-score (|z|>3): 정규분포 가정 필요 → 비정규/왜도↑에 취약
  • IQR (Q1−1.5×IQR, Q3+1.5×IQR): 왜도↑/비대칭 분포에도 강건
  • 수정 Z-score: 왜도·이상치에 더 Robust

다양한 이상치 탐지 방법들

📝 이상치 처리 방법
  • Winsorizing: 극단값을 분위수 경계값으로 대체
  • Capping: 도메인 기반 경계치로 제한
  • Trimming/Removal: 이상치 제거 (명백한 오류값일 때만)
  • Domain Knowledge: 임상 기준·공학 스펙 등 실무적 기준 반영

결측치 처리

단순 대체 방법

  • 평균 대체: 계산 매우 단순, 빠름 (왜도 분포에서는 왜곡 가능)
  • 중앙값 대체: 왜도↑ 분포에서 안정적, 분포 왜곡 최소화
  • 최빈값 대체: 범주형 변수에 자주 사용
  • 무작위 대체: 분산 보존하지만 데이터 일관성 낮음

결측치 처리의 다양한 방법들

예측 기반 대체 방법

  • KNN Imputation: 유사한 K개의 이웃을 찾아 평균/중앙값으로 대체
  • 회귀 Imputation: 다른 변수 기반 회귀 모델로 예측

데이터 전처리의 장점

  • 모델 성능 향상: 적절한 전처리로 모델 성능 크게 개선
  • 학습 안정성: 수렴 속도 향상 및 안정적인 학습
  • 해석 가능성: 전처리된 특성은 해석하기 쉬움
  • 일반화 성능: 과적합 방지 및 일반화 능력 향상
